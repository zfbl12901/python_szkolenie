# pipx

`pipx` permet d'installer et exécuter des applications Python dans des environnements isolés, sans polluer l'environnement global. C'est l'outil idéal pour installer des outils en ligne de commande.

## Concepts de base

**pipx** installe chaque application Python dans son propre environnement virtuel isolé. C'est parfait pour les outils en ligne de commande que vous voulez utiliser globalement sans créer de conflits avec vos projets.

### Différence avec pip

- **pip** : Installe des packages dans l'environnement actuel (venv ou global)
- **pipx** : Installe des applications dans des environnements isolés dédiés

```bash
# pip : installation dans l'environnement actuel
pip install black  # Installe dans venv/ ou globalement

# pipx : installation isolée
pipx install black  # Crée un environnement isolé pour black
```

## Installation d'applications

### Installation basique

```bash
# Installer une application
pipx install black

# L'application est maintenant disponible globalement
black --version

# pipx crée un environnement isolé pour black
# ~/.local/pipx/venvs/black/
```

### Installation avec version spécifique

```bash
# Installer une version spécifique
pipx install "black==22.0.0"

# Installer depuis Git
pipx install git+https://github.com/psf/black.git
```

### Applications courantes à installer avec pipx

```bash
# Outils de formatage et linting
pipx install black
pipx install ruff
pipx install mypy

# Outils de développement
pipx install cookiecutter
pipx install httpie
pipx install poetry

# Outils de build
pipx install build
pipx install twine
```

## Avantages par rapport à pip global

### Isolation complète

```bash
# ❌ pip global : conflits possibles
pip install black==22.0.0
pip install ruff  # Peut nécessiter une version différente de black

# ✅ pipx : isolation totale
pipx install black==22.0.0
pipx install ruff  # Environnements séparés, pas de conflit
```

### Disponibilité globale

```bash
# Avec pipx, les applications sont disponibles partout
pipx install black

# Utilisable depuis n'importe quel répertoire
cd /any/directory
black mon_fichier.py  # ✅ Fonctionne
```

### Pas de pollution de l'environnement

```bash
# ❌ pip global : pollue l'environnement
pip install black ruff mypy pytest
# Tous les packages dans le même environnement

# ✅ pipx : chaque outil isolé
pipx install black
pipx install ruff
pipx install mypy
# Chaque outil dans son propre environnement
```

## Cas d'usage

### Cas d'usage 1 : Outils de développement

```bash
# Installer des outils de développement globalement
pipx install black
pipx install ruff
pipx install mypy
pipx install pytest

# Utilisables partout
black format.py
ruff check .
mypy mon_module.py
```

### Cas d'usage 2 : Applications CLI

```bash
# Applications en ligne de commande
pipx install httpie  # Client HTTP
pipx install youtube-dl  # Téléchargeur YouTube
pipx install cookiecutter  # Générateur de projets

# Utilisation
http GET https://api.example.com
youtube-dl https://youtube.com/watch?v=...
cookiecutter template
```

### Cas d'usage 3 : Outils de build et publication

```bash
# Outils pour le packaging
pipx install build
pipx install twine

# Utilisation
python -m build
twine upload dist/*
```

## Exemples pratiques

### Exemple 1 : Installation et utilisation

```bash
# Installer black
pipx install black

# Vérifier l'installation
pipx list

# Utiliser black
black mon_fichier.py

# Mettre à jour
pipx upgrade black

# Désinstaller
pipx uninstall black
```

### Exemple 2 : Installation avec dépendances injectées

```bash
# Installer avec des dépendances supplémentaires
pipx install "black[colorama]"

# Ou injecter des packages dans l'environnement
pipx inject black colorama
```

### Exemple 3 : Exécution temporaire

```bash
# Exécuter une application sans l'installer
pipx run black mon_fichier.py
pipx run httpie GET https://api.example.com

# Utile pour tester avant d'installer
```

## Commandes essentielles

### Lister les applications installées

```bash
# Lister toutes les applications
pipx list

# Sortie
# package black 22.0.0, Python 3.9.0
#   - black
# package ruff 0.0.200, Python 3.9.0
#   - ruff
```

### Mettre à jour

```bash
# Mettre à jour une application
pipx upgrade black

# Mettre à jour toutes les applications
pipx upgrade-all
```

### Désinstaller

```bash
# Désinstaller une application
pipx uninstall black
```

### Informations

```bash
# Voir les informations d'une application
pipx list --include-injected

# Voir l'emplacement
pipx list --short
```

## Bonnes pratiques

### 1. Utilisez pipx pour les outils CLI

```bash
# ✅ Bon : outils CLI avec pipx
pipx install black
pipx install ruff
pipx install httpie

# ❌ Mauvais : outils CLI avec pip global
pip install black  # Pollue l'environnement global
```

### 2. Utilisez pip dans venv pour les dépendances de projet

```bash
# ✅ Bon : dépendances de projet avec pip dans venv
python -m venv venv
source venv/bin/activate
pip install requests flask

# pipx est pour les outils, pas pour les dépendances
```

### 3. Mettez à jour régulièrement

```bash
# ✅ Mettre à jour les outils
pipx upgrade-all
```

## Comparaison avec d'autres méthodes

### pipx vs pip global

```bash
# pip global : tous les packages ensemble
pip install black ruff mypy
# Risque de conflits, pollution

# pipx : isolation par application
pipx install black
pipx install ruff
pipx install mypy
# Pas de conflits, isolation complète
```

### pipx vs npm global

```bash
# Similaire à npm global
npm install -g create-react-app

# pipx équivalent
pipx install cookiecutter
```

## Points clés à retenir

- ✅ `pipx` installe des **applications** dans des **environnements isolés**
- ✅ Parfait pour les **outils CLI** que vous voulez utiliser globalement
- ✅ **Pas de conflits** : chaque application a son propre environnement
- ✅ **Disponibilité globale** : applications accessibles partout
- ✅ Utilisez `pipx` pour les **outils**, `pip` dans `venv` pour les **dépendances**
- ✅ `pipx run` permet d'**exécuter sans installer**
- ✅ `pipx upgrade-all` met à jour **toutes les applications**

pipx est l'outil idéal pour installer des applications Python en ligne de commande. Il combine la commodité d'une installation globale avec la sécurité de l'isolation.
